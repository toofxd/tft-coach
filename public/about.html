Tactician's Fundamentals — TFT Set 17
Most TFT tools give you stats. This one tries to explain what to do with them.
Why it exists

When I took Science and Strategy of AI at UCLA Anderson, I knew I wanted my final project to be a gaming application. TFT felt like the ideal use case: it has rich structured data, clear performance differences across skill levels, and no tool that actually explains why a Challenger player makes different decisions than everyone else. After learning about retrieval-augmented generation and how models can be grounded in domain-specific context, I wanted to build an AI that could reason about TFT the way a high-elo player does — rather than just repeat generic tier-list advice.

High-elo players don't win because they memorized patch notes. They've built intuition through thousands of games — how to manage economy (greed, roll, or level), how to preserve streaks, and how to itemize (slam or BSS). That knowledge usually lives in experience and VOD review, not in a system that can explain the gap back to a player in a structured way. This project is an attempt to close that gap — combining curated high-elo match data with LLM-based reasoning so the output feels less like a static stats page and more like a coach that can interpret what winning TFT actually looks like.

How it works

The system is built around a dataset of roughly 140,000 Challenger and Grandmaster TFT matches from North America, updated periodically. From that data, I compute benchmarks around leveling efficiency, economy management, placement outcomes, and item performance — with win-rate metrics anchored to top-4 finishes so the model is grounded in what successful play looks like, not what's merely common.

The reasoning layer uses a RAG-style approach. Instead of relying only on what a model may have seen during training, each response is grounded at inference time in a TFT-specific knowledge base — champion data, traits, item stats, and high-elo benchmarks — which keeps the system tied to current game context and structured evidence rather than generalized recall. Three models work together: Gemini 2.5 Flash handles coaching and live search, Groq LLaMA 3.3 handles champion-specific tips and fallback responses, and Riot's Match API provides personal game history when available.

What you can do here
  • Stat Calculator — check a unit's stats across star levels and item combinations, with recommendations derived from Challenger win-rate data
  • TFT Coach — enter your Riot ID to compare your economy and leveling decisions against high-elo benchmarks and see where your play diverged
  • Gods — full reference for all Set 17 god boons and trials, organized by god and stage
  • Ask AI — open-ended questions about comps, augments, and positioning, answered with live search grounding and a structured TFT reference layer
Data and limits

The Riot API provides an end-of-game snapshot for each match — enough to analyze board strength, itemization, placements, and outcomes at a meaningful level, but not a full reconstruction of every in-game decision. That's sufficient for personal match analysis, but falls short of what a true replay or event telemetry feed would allow.

Pro-play analysis is similarly constrained. Ranked match patterns can be quantified from structured data, but tournament and top-level decision-making still requires leaning partly on VODs and community interpretation rather than a clean official data layer.

Personal game lookup can break when the Riot developer key expires — development keys require manual renewal on a short cycle. A production API key would be the clearest next step toward making that part of the product reliable. Champion and item references also depend on Community Dragon and Riot Data Dragon, so not every number is guaranteed to be fully current with the latest patch.

Champion and item data is sourced from Community Dragon and Riot Data Dragon. Win rates update when new match data is collected. Check patch notes for recent changes.
