## Daily Data Collection

When the user says "run daily collection", execute the scheduled task:
```powershell

```
Then tail the log to show progress: `Get-Content logs\collect_$(Get-Date -Format 'yyyy-MM-dd').log -Wait`

The task runs `collect_pro.py ‚Üí features.py ‚Üí train.py` and logs to `logs/collect_YYYY-MM-DD.log`.

## Architecture Overview

### Server
- `server.js` ‚Äî Express on port `30002`, binds to `127.0.0.1` in local dev, `0.0.0.0` on Railway
- Serves static files from `/public`
- Riot API key, Gemini key, and Groq key loaded from `.env`

### Data Pipeline (run manually to refresh data)
```
src/collect_pro.py       ‚Üí scrapes Challenger/GM/Master match data via Riot API
src/features.py          ‚Üí processes matches into CSV features
src/train.py             ‚Üí trains placement model
src/build_calculator_data.py  ‚Üí fetches tactics.tools + CommunityDragon ‚Üí data/static/calculator_data.json
src/build_calculator.py  ‚Üí injects calculator_data.json into public/calculator.html
```

### After running `build_calculator_data.py` or `build_calculator.py`
The script writes to `data/static/calculator_data.json` and injects it into `public/calculator.html`.
**Important:** DATA injection uses string split, NOT `re.sub` ‚Äî regex replacement corrupts `\n` escape sequences in JSON and breaks the JS:
```python
pre  = html.split('const DATA = ')[0]
post = html.split('const UNITS_LIST')[1]
new_html = pre + 'const DATA = ' + data_json + ';\nconst UNITS_LIST' + post
```

## Pulling More Raw Data from Riot API

### How deduplication works
`collect_pro.py` has two layers of dedup ‚Äî no risk of double-counting even if re-run:
1. **Match ID dedup:** all IDs from all players are collected into a `set()` before any fetching
2. **File cache dedup:** before fetching a match, it checks if `data/raw/matches/{match_id}.json` already exists and skips it if so

So re-running the script at any time is always safe ‚Äî it only fetches genuinely new matches.

### Current tiers collected
The default in `collect()` is `tiers=("challenger", "grandmaster")` ‚Äî both are already included.
Current dataset: ~79,700 rows in `data/processed/match_features.csv`, ~12,900 rows in `item_winrates.csv`.

### To add Master tier
Edit line 84 of `src/collect_pro.py`:
```python
# Before
tiers: list[str] = ("challenger", "grandmaster"),

# After
tiers: list[str] = ("challenger", "grandmaster", "master"),
```
Master has 1,000‚Äì2,000+ players in NA. The `max_players=100` cap keeps it manageable.
Estimated time on a **dev API key**: 30‚Äì90 min (rate limited to ~50 req/min with backoff).

### To increase sample size per tier
Raise `max_players` (default 100) and/or `matches_per_player` (default 20):
```python
collect(
    tiers=("challenger", "grandmaster", "master"),
    matches_per_player=30,   # more history per player
    max_players=200,          # more players per tier
)
```
Be aware: dev keys are 100 req/2min. For large runs, expect multi-hour runtimes.
A **production API key** (applied for at developer.riotgames.com) removes this bottleneck.

### Full refresh sequence after collecting new data
```powershell
# 1. Disable sleep and hibernate first (hibernate requires elevated/admin PowerShell)
powercfg /change standby-timeout-ac 0
powercfg /hibernate off

# 2. Run collection (detached so it survives tool timeouts)
Start-Process -FilePath "python" -ArgumentList "src/collect_pro.py" -NoNewWindow -RedirectStandardOutput "collect_log.txt" -RedirectStandardError "collect_err.txt"

# 3. Monitor progress
Get-Content collect_log.txt -Tail 5

# 4. Once done, re-enable sleep and hibernate (hibernate requires admin PowerShell)
powercfg /change standby-timeout-ac 30
powercfg /hibernate on

# 5. Rebuild data and restart server
python src/features.py       # rebuild match_features.csv + item_winrates.csv
python src/train.py          # retrain model
node server.js               # restart server to reload CSVs
```

‚öÝÔ∏è **Do not shut down or hibernate** during collection ‚Äî locking the screen is fine.
- Sleep ‚Üí disabled via `powercfg` above, won't trigger automatically
- Hibernate ‚Üí disable manually in **admin** PowerShell: `powercfg /hibernate off`
- Shut down / restart ‚Üí kills the process, must restart collection

### Raw match files location
`data/raw/matches/*.json` ‚Äî one file per match ID. Safe to inspect individually.
Do not delete these ‚Äî they are the cache that prevents re-fetching.

## Known Issues & Fixes Applied

### Riot API rate limiting
`Promise.all` with 200 simultaneous match detail requests kills most with 429 silently.
**Fix:** `batchMatchDetails()` fetches in groups of 10 with 1.1s delay. `allCount` set to 50.

### Play rate > 100%
Items duplicated in combos (e.g. `[Deathblade, Deathblade, LW]`) counted twice.
**Fix:** `new Set(combo.items)` in `/api/item-recs`.

### DATA injection corruption
Using `re.sub()` to inject JSON into HTML expands `\n` sequences into real newlines ‚Üí JS syntax error.
**Fix:** String split approach (see above).

### filterDrop ReferenceError
`function filterDrop(type)` ‚Äî must have `autoOpen` as second param or all dropdowns break.
**Fix:** Always keep signature as `function filterDrop(type, autoOpen)`.

## Pro Dataset Integration (server.js)

All AI prompts are grounded in the Challenger/GM dataset. **Only top-4 (winning) games are used for efficiency/economy benchmarks** ‚Äî losing games are excluded so the AI models what success looks like, not the average.

### Pre-computed at startup
- **`challengerBench`** ‚Äî computed once from `match_features.csv` on server start:
  - `avgLevelEff`, `avgGoldLeft`, `avgLevel` ‚Üí from top-4 rows only
  - `top4Rate` ‚Üí winners / all rows (needs full denominator)
  - Injected into `/api/coaching` and `/api/coach` prompts as Challenger/GM reference numbers

### Per-request helpers
- **`proStats(topTrait)`** ‚Äî filters `matchFeatures` to rows matching the comp's `top_trait`, then computes efficiency/gold stats from top-4 rows only. Used by `/api/coach` for comp-specific benchmarks.
- **`proUnitStats(unitId)`** ‚Äî queries `itemWinrates` for the champion (min 5 games), returns top 4 item combos with actual win % and game counts. Injected into `/api/champion-tips` prompt. `top4_rate` in `itemWinrates` is already a win-rate ratio, so no filtering needed here.

### What each AI endpoint receives from the dataset
| Endpoint | AI routing | Pro data injected |
|----------|------------|-------------------|
| `/api/champion-tips` | Groq only (intentional) | Top 4 item combos with win % + game count from `itemWinrates` |
| `/api/coach` | `callAI` ‚Äî Gemini ‚Üí Groq fallback | Comp-specific avgLevelEff, avgGoldLeft, top4Rate (top-4 games only) + overall `challengerBench` |
| `/api/coaching` | `callAI` ‚Äî Gemini ‚Üí Groq fallback | Overall `challengerBench` (avgLevelEff, avgGoldLeft, avgLevel, top4Rate) |
| `/api/ask` | `callGeminiWithSearch` ‚Üí Groq fallback | Champion roster, trait breakpoints, craftable items (see below) + live Google Search grounding via Gemini |

## Ask AI Prompt Format (`/api/ask`)

Free-form Q&A is the highest hallucination-risk endpoint ‚Äî the AI is grounded two ways:

### 1. Static grounding sheets (injected into every prompt)
Built from `data/static/calculator_data.json` at server startup:
- **`champSheet`** ‚Äî all 67 champions: name, cost, traits, role (one line each)
- **`traitSheet`** ‚Äî all 35 traits with their valid breakpoints only (e.g. `Meeple (3/5/7/10)`)
- **`itemSheet`** ‚Äî all 39 craftable items with primary stat bonuses (HP/AD/AP/armor/MR/mana regen). Raw engine variables (AS multipliers, crit, etc.) are excluded to avoid garbage in the prompt.

Traits are deduplicated by name (keep entry with most breakpoints). `Choose Trait` and traits with no breakpoints are excluded.

### 2. Live Google Search grounding (Gemini only)
`callGeminiWithSearch` adds `"tools": [{"google_search": {}}]` to the Gemini API request. Gemini automatically searches when needed (e.g. augments, patch notes, anything not in the static sheets). Billed per prompt ‚Äî **5,000 free/month**, then $14/1,000.

### Fallback behavior
If Gemini fails or is rate-limited, Groq answers using only the static grounding sheets (no web search). The prompt instructs both models to flag anything not covered by the sheets as *"based on my training data, which may be outdated."*

## Champion Tips Prompt Format (`/api/champion-tips`)

The Groq prompt for champion tips enforces a specific output format ‚Äî **do not change this without re-testing.**

**Best Items** section rules:
- First sentence must follow: `"[Name] benefits from [Item A] for [reason A], [Item B] for [reason B], and [Item C] for [reason C]."`
- Reasons are short (2‚Äì5 words)
- If the champion already has strong base AD/AP/survivability, the tip should note they need to cover a gap (shred, burn, wound) or stack their primary stat
- Mention **sunder/armor shred** (Last Whisper), **burn** (Sunfire Cape / Red Buff), or **grievous wounds** (Morellonomicon / Red Buff) when relevant to the champion's damage type

## UI Structure

Three pages sharing the same tab bar: **Stat Calculator**, **TFT Coach** (`public/calculator.html`), **Gods** (`public/gods.html`).

### Calculator tab (`#tab-calculator`)
- Left column: champion search, cost filter buttons (`setCostFilter` clears unit input), item/trait selectors
- Right column: stat table, item recommendations (icon grid with hover tooltips), champion tips card (`#tipsCard`)
- `selectOpt(key, unit)` ‚Üí `fetchItemRecommendations(key).then(() => fetchChampionTips(unit, key))` (sequential ‚Äî tips need item recs loaded first; `key` is the raw `TFT17_X` unit ID passed to the server for pro data lookup)
- `filterDrop(type, autoOpen)` ‚Äî `autoOpen=false` suppresses dropdown on cost filter clicks

### Coach tab (`#tab-coach`)
- Riot ID input ‚Üí `loadProfile()` ‚Üí `/api/profile`
- All Set 17 stats banner (`#coachStats`) populated from `profileData.allStats`
- Recent N games banner (`#coachRecentStats`) populated from `profileData.recentStats`
- AI coaching box populated from `/api/coaching`
- Ask AI box (`#askInput`) ‚Üí `/api/ask` ‚Üí Gemini+Search (Groq fallback)

### Markdown formatting
`formatCoaching()` and `formatTips()` ‚Äî bold blue headers from `**text:**` patterns, used for AI responses.
